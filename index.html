1. 線性代數 — 向量、矩陣、內積

神經網路的每一層，本質就是矩陣乘法 + 非線性轉換。先把語言備齊。

1.1 向量與內積 — 投影與量測對齊程度

一個 $n$ 維向量 $\mathbf{x}=(x_1,x_2,\dots,x_n)^\top$ 可看成一個資料點的特徵（例如一條 flow 的封包數、bytes、時長…）。內積 (dot product) 是最核心的運算：

$$\mathbf{w}\cdot\mathbf{x} = \mathbf{w}^\top\mathbf{x} = \sum_{i=1}^{n} w_i x_i$$

它衡量兩向量的「對齊程度」。幾何上 $\mathbf{w}^\top\mathbf{x} = \lVert\mathbf{w}\rVert\,\lVert\mathbf{x}\rVert\cos\phi$，其中 $\lVert\mathbf{x}\rVert=\sqrt{\sum_i x_i^2}$ 是長度、$\phi$ 是夾角。一個神經元做的就是一次內積 $\mathbf{w}^\top\mathbf{x}$ 加偏置 $b$，代表「用權重量測輸入有多符合某樣式」。（注：這裡的「對齊」是方向與長度的綜合分數；純方向的相似度需先正規化，見 1.5。）

幾何意義。 內積 = 「$\mathbf{a}$ 投影到 $\mathbf{b}$ 方向的長度 $\lVert\mathbf{a}\rVert\cos\phi$」再乘上 $\lVert\mathbf{b}\rVert$。夾角越小、投影越長、內積越大。

值怎麼解讀。 內積不是「大 / 小」兩極，而是一條有正負的軸 — 越正越對齊、近 0 不相關、越負越相反：

內積值夾角 φcos φ對齊狀態
最大（正）0°+1完全同向（最對齊）
≈ 090°0垂直、不相關
最小（負）180°−1完全反向（最不對齊）

注意：內積 $=\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi$，值大也可能只是向量長；要純比方向對齊、不受長度干擾，看 $\cos\phi\in[-1,1]$（正規化後的內積，見 1.5）。神經元同理：輸入越對齊權重樣式，內積越大、激活越強（負值＝強烈不符合）。

內積的代數定義 $\sum_i a_ib_i$ 與幾何定義 $\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi$ 為何會相等？關鍵是餘弦定律；完整推導見附錄 A。

1.2 神經元 — 把線性代數變成「會學習的特徵偵測器」

人工神經元是深度學習的最小單元，靈感來自生物神經元（收多個輸入、加權整合、超過門檻才「激發」放電）。它的數學定義只有兩步：

$$z = \mathbf{w}^\top\mathbf{x} + b = \sum_i w_i x_i + b, \qquad a = \sigma(z)$$

  • $\mathbf{x}$：輸入（上一層輸出或原始特徵）
  • $\mathbf{w}$：權重（可學；代表「這神經元在找什麼樣式」）
  • $b$：偏置（可學；調整激發門檻）
  • $\sigma$：激活函數（非線性，如 ReLU / sigmoid，見主題 9）
  • $a$：輸出（激活值）

權重 $\mathbf{w}$ — 特徵樣板。 $\mathbf{w}^\top\mathbf{x}$ 是內積（1.1），量測輸入與「樣式 $\mathbf{w}$」的對齊程度。所以權重就是這個神經元學到的特徵樣板：輸入越像 $\mathbf{w}$、內積越大、越可能激發 — 這就是「神經元＝特徵偵測器」的由來。

權重從哪來？ 上面的 $\mathbf{w}, b$ 不是預設的，而是訓練學出來的：初始隨機，再用梯度下降 + 反向傳播依損失反覆微調（見主題 3 / 6 / 10）。幾何上，訓練就是一直挪動決策邊界（調整法向量 $\mathbf{w}$ 與偏置 $b$），從隨機初始逐步把不同類別分到正確的兩側；因為內積在對齊時最大，$\mathbf{w}$ 會自然長成「這神經元專門偵測的樣式」——這就是「特徵樣板」的由來（資料 + 損失驅動的表徵學習）。

偏置 $b$ — 門檻。 幾何上 $\mathbf{w}^\top\mathbf{x}+b=0$ 是輸入空間裡的一個決策超平面（2D 是一條線、3D 是平面），$\mathbf{w}$ 是它的法向量。$b$ 決定這平面離原點多遠、即「多容易被激發」。$z=\mathbf{w}^\top\mathbf{x}+b$ 可看成輸入點到超平面的帶符號距離（乘上 $\lVert\mathbf{w}\rVert$）：一側為正（激發）、另一側為負（不激發）。

激活 $\sigma$ — 非線性轉換。 若沒有 $\sigma$，多層神經元疊起來仍是線性（矩陣連乘 $\mathbf{W}_2\mathbf{W}_1\mathbf{x}$ 還是一個矩陣），疊幾層都只能學線性邊界。加入非線性轉換後，多層才能逼近任意複雜函數 — 這是深度學習能學複雜樣式的關鍵（激活函數細節見主題 9）。

接回上下文。 一個神經元 = 一次內積 + 偏置 + 非線性轉換；一層 = 很多神經元 = 一個矩陣 $\mathbf{W}$（每列是一個神經元的權重，即 1.3 的列視角）：$\mathbf{z}=\mathbf{W}\mathbf{x}+\mathbf{b}$、$\mathbf{a}=\sigma(\mathbf{z})$；多層堆疊 + 非線性轉換 = 深度網路，前一層的輸出當下一層輸入，逐層抽取越來越抽象的特徵。

1.3 矩陣乘法 — 座標變換，也是對各樣板的對齊程度

把一層 $m$ 個神經元的權重疊成矩陣 $\mathbf{W}\in\mathbb{R}^{m\times n}$，一次算出整層加權和：

$$\mathbf{z} = \mathbf{W}\mathbf{x} + \mathbf{b},\qquad z_j = \sum_{i=1}^{n} W_{ji}x_i + b_j$$

$\mathbf{z}\in\mathbb{R}^m$ 是這層的原始輸出 (pre-activation)。一次前向傳播就是連串的 $\mathbf{W}\mathbf{x}+\mathbf{b}$。這也是為什麼 GPU/NPU 有用 — 它們為大規模矩陣乘法 (GEMM) 而生。

維度規則（怎麼判斷結果形狀）。 矩陣 × 向量得到的是向量（不是矩陣）：

$$\mathbf{W}_{m\times n}\,\mathbf{x}_{n\times 1} = \mathbf{z}_{m\times 1}$$

內側維度 $n$ 必須相等（W 欄數 = x 維度）；結果是 $m\times 1$ 欄向量，長度＝W 的列數 $m$（＝該層神經元數）。因為每個 $z_j$ 是「W 第 $j$ 列 $\cdot$ x」的內積（純量），$m$ 列 → $m$ 個純量 → $m$ 維向量。

運算維度結果
矩陣 × 向量$(m\times n)(n\times 1)$向量 $m\times 1$
矩陣 × 矩陣$(m\times n)(n\times p)$矩陣 $m\times p$
向量ᵀ × 向量（內積）$(1\times n)(n\times 1)$純量
向量 × 向量ᵀ（外積）$(n\times 1)(1\times n)$矩陣 $n\times n$

（向量本身即 $m\times 1$ 矩陣；矩陣 × 向量是矩陣乘法的特例，結果剛好「一欄」故稱向量。）

幾何意義。 矩陣乘一個向量 = 對整個空間做一次線性變換（旋轉 / 縮放 / 剪切 / 翻轉）。最直覺的看法：矩陣的每一「欄」就是原本的座標軸（基底）被變換後落到的位置。因為

$$\mathbf{W}\mathbf{x} = x_1\,\mathbf{w}_1 + x_2\,\mathbf{w}_2 + \dots + x_n\,\mathbf{w}_n$$

其中 $\mathbf{w}_j$ 為 $\mathbf{W}$ 的第 $j$ 欄。$\mathbf{W}\mathbf{x}$ 就是「用 $\mathbf{x}$ 的分量，把 $\mathbf{W}$ 的各欄（變換後的軸）加權相加」。由基底張成的單位正方形，會被 $\mathbf{W}$ 拉成一個平行四邊形，其邊就是 $\mathbf{W}$ 的欄向量：

兩種互補視角：

  • 欄視角（空間變換）：欄＝基底落點；$\mathbf{W}\mathbf{x}$＝$\mathbf{x}$ 被變換後的新位置。
  • 列視角（特徵偵測）：每一列是一個權重向量，$z_j=\mathbf{w}_{j,:}^\top\mathbf{x}$ 是內積＝$\mathbf{x}$ 對第 $j$ 個樣式的對齊分數 → 一層＝一排特徵偵測器（接回 1.2 神經元）。把這個「特徵偵測器」在影像上到處滑動掃描，就是卷積（主題 12）。

矩陣相乘＝變換的合成：$\mathbf{A}(\mathbf{B}\mathbf{x})$ 是「先做 B 再做 A」。特殊情形：對角矩陣＝沿各軸縮放（接回 1.6 特徵值）、旋轉矩陣＝轉一個角度、行列式＝面積 / 體積被放大的倍率。

行列式 (determinant)：如果矩陣是方陣，「行列式」就是可以把方陣轉變成一個數的運算：2×2 時 $\det\begin{bmatrix}a&b\\c&d\end{bmatrix}=ad-bc$。幾何意義：$|\det|$ 就是這個變換把「大小」放大的固定倍數（2D 的大小是面積、3D 是體積、n 維是 n 維體積）。以 2D 為例：基底張成的單位正方形（面積 1）經矩陣拉成一個平行四邊形，這個平行四邊形的面積就是 $|\det|$。特例：$\det=1$ 面積不變、$\det\lt0$ 方向被翻轉（鏡射）、$\det=0$ 代表把空間「壓扁」到更低維（面積歸零）、矩陣不可逆。這個「$\det=0$＝壓扁空間」在 1.6 解特徵值時會關鍵地用到。

用欄還是列？矩陣＝欄向量並排。 一個 2×2 矩陣可看成兩個欄向量並排，$\det$ 就是這兩個欄向量張成的平行四邊形面積。改用列向量算數值一樣（因為 $\det(\mathbf{A})=\det(\mathbf{A}^\top)$）；只是講「變換把空間拉成平行四邊形」的幾何時慣例用欄向量（＝基底被變換後搬到的位置）。以下用實例 $\mathbf{A}=\begin{bmatrix}3&1\\2&3\end{bmatrix}$ 看「拉成平行四邊形」的完整過程。

把單位正方形的 4 個角逐一搬過去（這就是「拉成」的字面意思）。 基底 $\mathbf{e}_1=(1,0)$、$\mathbf{e}_2=(0,1)$ 張成一個 1×1 的單位正方形（面積 = 1，當量尺）；它的四個角經 $\mathbf{A}$（$\mathbf{A}\begin{bmatrix}x\\y\end{bmatrix}=\begin{bmatrix}3x+y\\2x+3y\end{bmatrix}$）搬到新位置：

原本的角經 A 搬到
$(0,0)$$(0,0)$
$(1,0)$（$\mathbf{e}_1$）$(3,2)$ ← A 的第 1 欄
$(0,1)$（$\mathbf{e}_2$）$(1,3)$ ← A 的第 2 欄

四個新位置連起來就是下圖的黃色平行四邊形；面積從 1 變成 7 → 放大 7 倍 → $\det\mathbf{A}=7$。

3D 版：單位立方體 → 平行六面體。 換成 3×3 矩陣，它作用在 3D 空間：三個欄向量張成一個平行六面體 (parallelepiped)，det = 它的體積。單位立方體（體積 1）被拉成這個平行六面體，體積就是 |det|。例 $\mathbf{A}=\mathrm{diag}(2,3,1)$：把立方體拉成 $2\times3\times1$ 的箱子，體積 $=2\times3\times1=6=\det\mathbf{A}$；det = 0 則壓扁成平面或線（體積 0）。同一個「大小放大的固定倍數」概念，2D 是面積、3D 是體積、n 維是 n 維體積。

行列式的應用。 它遠不只算面積，是線代的常用工具：

  • 可逆性／線性相關（最常用的開關）：det ≠ 0 → 可逆 — 變換一對一、沒壓扁空間，能「倒帶」還原（存在反矩陣 $\mathbf{A}^{-1}$，$\mathbf{A}^{-1}\mathbf{A}=\mathbf{I}$），欄向量線性獨立；det = 0 → 不可逆（奇異） — 空間被壓扁、不同輸入被壓到同一輸出（資訊丟失、還不回去），欄向量線性相關。它決定 $\mathbf{A}\mathbf{x}=\mathbf{b}$ 有沒有唯一解（回歸、共線性、正則化的根源）。
  • 解線性方程組：$\mathbf{A}\mathbf{x}=\mathbf{b}$，det ≠ 0 有唯一解、det = 0 無解或無限多（克拉瑪公式直接用 det 解）。
  • 求特徵值：$\det(\mathbf{A}-\lambda\mathbf{I})=0$（見 1.6，PCA／譜分析入口）。
  • 幾何：三角形／多邊形面積、體積、判斷共線共面（det = 0）、方向（det 正負＝有無翻轉／鏡射）。
  • 機率／ML：多元高斯分布的協方差 $\det\boldsymbol{\Sigma}$（機率密度歸一化常數）；換座標的 Jacobian 行列式（多重積分換元、生成模型 normalizing flows 用 $\log\lvert\det\mathbf{J}\rvert$）。

非方陣呢？ 非方陣（列數 ≠ 欄數）沒有行列式 — 因為它「進出不同維度」，沒有單一的體積倍率。要量體積改用 Gram 行列式 $\sqrt{\det(\mathbf{A}^\top\mathbf{A})}$、要通用縮放用奇異值 (SVD)、要求逆用偽逆 (pseudoinverse)。ML 常把非方陣的資料矩陣 $\mathbf{X}$（$m$ 筆 × $n$ 特徵）湊成方陣 $\mathbf{X}^\top\mathbf{X}$ 來用（回歸的正規方程就是這樣）。

1.3.1 一層的輸出在幾何上是什麼？（換座標系＋非線性摺疊）

前面談的是「一層的線性部分 $\mathbf{W}\mathbf{x}+\mathbf{b}$」。但真正餵給下一層的，是再經過激活 $\mathbf{a}=\sigma(\mathbf{z})$ 的輸出。那個輸出向量 $\mathbf{a}$ 在幾何上代表什麼？由淺到深有四層意義：

  1. 每個分量＝一次投影（對齊程度）。 分量 $z_j=\mathbf{w}_j\!\cdot\!\mathbf{x}+b_j$ 的幾何意義（1.1）是把輸入點 $\mathbf{x}$ 投影到方向 $\mathbf{w}_j$，量「$\mathbf{x}$ 有多對齊第 $j$ 個學到的特徵樣板」；等價地，$z_j$ 正比於 $\mathbf{x}$ 到「第 $j$ 個神經元決策超平面」的帶號距離。
  2. 整個輸出向量＝資料點在「新特徵座標系」的新座標（re-embedding）。 原本 $\mathbf{x}$ 用原始特徵軸描述；一層之後改用「對樣板 1 的對齊度、對樣板 2 的對齊度…」這組新軸來描述同一個點的位置。所以一層的輸出＝那個點在「由學到的方向張成的新空間」裡的新座標；下一層看到的不是原始資料，而是這份重新表達過的座標。
  3. 非線性 $\sigma$＝把空間摺疊／扭曲（這是「深度」有效的關鍵）。 若沒有 $\sigma$，多層只是矩陣連乘＝仍是一個矩陣（1.3），幾何上只有旋轉＋縮放＋剪切，再多層也切不開本來線性不可分的資料。$\sigma$（ReLU、sigmoid）會把座標空間摺起來、壓扁、彎折，於是原本纏在一起的兩類被拉到「一刀（下一層超平面）就能分開」的位置。
  4. 維度變化＝升維展開／降維壓縮。 神經元數 > 輸入維度 → 升維攤開（類似 kernel trick，本來擠一起的更好分）；神經元數 < 輸入維度 → 降維壓縮，只留最有判別力的特徵座標。

摺紙比喻。 一張畫著兩條交纏螺旋的紙，攤平時一條直線分不開；把紙摺幾摺後，一刀就能切開。每一層的輸出，就是那張紙被摺過一次後的新狀態。

一句話總結：前一層的輸出＝同一個資料點被重新表達成「在一組學到的方向上各投影多少」的新座標，而且整個空間已被非線性摺過一次，好讓下一層的超平面更容易把類別切開。這就是「深度」的意義——每多一層，就多摺一次、換一次更有利的座標系（接回 1.2 神經元、前接主題 8 前向傳播）。

1.4 常用範數與相似度

先從「長度」與「距離」的幾何直覺講起。 一個向量可看成從原點指向某點的箭頭。最自然的兩個量是：長度＝這支箭頭有多長（原點到那個點多遠）；距離＝兩個點相隔多遠，也就是差向量 $\mathbf{a}-\mathbf{b}$ 的長度。日常算長度用的是直線距離（畢氏定理）$\sqrt{x_1^2+x_2^2+\dots}$ — 鳥飛的最短直線，幾何上是一條直線段。

但「直線距離」不是唯一合理的量法。 在某些情況，用直線距離來比較並不恰當或不方便，需要換一種「怎麼算長度」的規則：

  • 市區棋盤格街道不能走對角線、只能沿街轉直角 → 實際距離是走過的格子總長，不是直線。
  • 有時只在乎變化最大的那一維（例如「每個規格都不能超標」）→ 該用「取最大那一維」當距離。
  • 資料有離群值，或想只看方向不看長短（比主題像不像）→ 直線距離都不是最好的選擇。

也就是說，「長度／距離」其實是一種可以選擇的定義 — 依你想比較什麼、在乎什麼而不同。為了能在後面公平地比較、挑到適合的度量，數學家把「一種合理的長度量法」抽象出來，取名叫範數 (norm)；同一組向量換一個範數，就換一種「怎麼算長度／距離」的規則。

範數的定義。 一種「長度量法」 $\lVert\cdot\rVert$（把向量映到非負實數）要夠格叫範數，必須滿足三條公理：

  • 非負性：$\lVert\mathbf{x}\rVert\ge 0$，且 $=0\iff\mathbf{x}=\mathbf{0}$（只有零向量長度為 0）。
  • 齊次性：$\lVert\alpha\mathbf{x}\rVert=|\alpha|\,\lVert\mathbf{x}\rVert$（放大幾倍、長度就幾倍）。
  • 三角不等式：$\lVert\mathbf{x}+\mathbf{y}\rVert\le\lVert\mathbf{x}\rVert+\lVert\mathbf{y}\rVert$（走直線最短，繞路只會更長）。

最常用的是 p-範數 $\lVert\mathbf{x}\rVert_p=\left(\sum_i|x_i|^p\right)^{1/p}$：

範數定義量距離的方式主要用途
L2（歐氏）$\sqrt{\sum x_i^2}$直線距離（畢氏定理）長度、距離、L2 正則化
L1$\sum|x_i|$沿座標軸走（計程車／曼哈頓）稀疏性、L1 正則化
L∞$\max_i|x_i|$只看變化最大的那一維最壞情況界限

幾何意義。 範數＝「原點到點 $\mathbf{x}$ 的距離」，但「距離」怎麼算隨範數而不同。最能看懂差異的是單位球：所有「長度 = 1」的點連成的形狀 — L2 是圓、L1 是菱形（尖角在座標軸上）、L∞ 是正方形。

先破除常見誤會：這三個形狀不是「一個向量的樣子」，而是「所有長度 = 1 的向量」連起來的集合（無限多個點，不是一個）。單一向量的長度只是一個數字；但不同範數對「長度 = 1」的認定不同，所以「長度都算 1 的那群點」連出來的形狀就不同。看同一批點在三種範數下的長度：

點 $(x_1,x_2)$L2 $\sqrt{x_1^2+x_2^2}$L1 $\lvert x_1\rvert+\lvert x_2\rvert$L∞ $\max$
$(1,0)$111
$(1,1)$1.4121
$(0.5,0.5)$0.7110.5

反過來問「哪些點的長度剛好 = 1？」三種範數的答案（點集）形狀就不同：L2 滿足 $x_1^2+x_2^2=1$ → 圓；L1 滿足 $\lvert x_1\rvert+\lvert x_2\rvert=1$ → 菱形（$(0.5,0.5)$ 在菱形上、卻不在圓上）；L∞ 滿足 $\max=1$ → 正方形（角落 $(1,1)$ 在正方形上，但 L2 長度是 1.41、不在圓上）。所以形狀是「這個範數認為哪些點跟原點一樣遠」的地圖，三者在 4 個座標軸點重合。

餘弦相似度 — 用範數把長度除掉。 回想 1.1：$\mathbf{a}\cdot\mathbf{b}=\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi$，內積混了兩件事 — 方向多對齊（$\cos\phi$）與兩者多長（$\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert$）。只看內積會被長度干擾（很長的向量就算方向沒多準、內積也大）。把內積除以兩者長度（範數），長度因子直接約掉，只剩方向：

$$\cos\phi=\frac{\mathbf{a}\cdot\mathbf{b}}{\lVert\mathbf{a}\rVert\,\lVert\mathbf{b}\rVert}=\frac{\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi}{\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert}\in[-1,1]$$

只跟夾角有關、與長度無關。例子：

兩向量內積（原始）餘弦相似度
$\mathbf{a}=(2,0),\ \mathbf{b}=(10,0)$（同方向、b 長 5 倍）$20$（看似很大）$\frac{20}{2\times10}=1$（方向完全相同）
$\mathbf{a}=(2,0),\ \mathbf{c}=(0,2)$（垂直）$0$$0$（不相關）

為什麼檢索要這樣：長文件與短文件若講同一主題，embedding 方向相近但長度差很多；語意檢索要判斷「同不同主題」，不該被長短干擾 → 用只看方向的餘弦相似度，而非會被長度灌水的原始內積（Embedding 語意檢索的核心，主題 15）。

範數的用途（不只正則化）。 範數是「量大小／長度／距離」的通用地基，貫穿整個教材：

應用用範數做什麼
距離 / 相似度兩點距離 $\lVert\mathbf{a}-\mathbf{b}\rVert$ → KNN、分群、異常偵測
正則化 (regularization)懲罰權重 $\lVert\mathbf{w}\rVert$ 防過擬合（L1 稀疏／L2 平滑，主題 7）
正規化 (normalization)$\mathbf{x}/\lVert\mathbf{x}\rVert$ 做單位向量、Layer/Batch Norm（1.5、主題 11）
餘弦相似度除以範數消掉長度 → Embedding 語意檢索（主題 15）
損失函數MSE ＝殘差的 L2 範數平方 $\lVert\hat{\mathbf{y}}-\mathbf{y}\rVert^2$（主題 4）
梯度裁剪 (gradient clipping)限制梯度大小 $\lVert\nabla\rVert$ 穩定訓練（主題 6）

所以範數是地基，正則化只是蓋在上面的其中一間房。易混提醒：正則化 = regularization（防過擬合）、正規化 = normalization（調尺度），只差一字、英文也像，做的事卻完全不同（見 1.5）。

為什麼不是只用 L2？不同範數配不同問題。 沒有一個範數「萬用」——選哪個，等於宣告「我認為怎樣算近、什麼該忽略」。L2 是好用的預設（平滑可微、對應直線距離與內積角度），但下列情況換別的更好：

情況L2 的問題改用
有離群值誤差被平方放大 → 少數極端點主宰結果L1（絕對值、線性，較穩健；如中位數 vs 平均）
要稀疏／選特徵圓沒角、不歸零L1（菱形尖角 → 權重歸零）

其中「離群值」最直觀：$\sum(\text{誤差})^2$ 裡誤差 10 貢獻 100、誤差 1 只貢獻 1，L2 會為了那個 10 拼命調整、忽略其他點；L1 用 $\sum|\text{誤差}|$，不被單一離群點綁架。另外「餘弦」這種角度型相似度是 L2 專屬（角度／內積綁在 L2 幾何，柯西–施瓦茲才保證 $\cos\phi\in[-1,1]$、相同方向剛好 = 1）；L1／L∞ 表達相似是靠「距離」而非角度（要相似度就把距離反過來，如 $1/(1+d)$）。這就是「沒有免費的午餐」：度量要配問題，不是 L2 打天下。

1.5 正規化 (normalization) — 把數值調到可比較的基準

正規化的精神：拿掉「不該影響判斷的量」，讓比較公平、訓練穩定。基礎有兩種（第三種在主題 11）：

(a) 向量正規化。 把向量除以自身長度，變成單位向量（長度 = 1）：

$$\hat{\mathbf{x}} = \frac{\mathbf{x}}{\lVert\mathbf{x}\rVert}$$

去掉長度、只留方向。正規化後，兩向量的內積就直接等於餘弦相似度：$\hat{\mathbf{a}}^\top\hat{\mathbf{b}}=\cos\phi$。幾何上這也代表「$\mathbf{a}$ 在單位向量 $\hat{\mathbf{b}}$ 上的投影長」。這就是為什麼 Embedding 語意檢索要正規化 — 比「像不像」不該被向量長短干擾（長文件與短文件講同主題，方向相近但長度差很多）。

(b) 特徵正規化。 訓練前把各特徵縮到可比範圍，否則大尺度特徵（如 bytes 百萬級）會壓過小特徵（如封包數幾十），主宰距離與梯度：

(c) 網路內正規化（主題 11 詳述）。 Batch/Layer Normalization 正規化各層激活值以穩定深層訓練；softmax 本身也是把分數正規化成「總和為 1」的機率分布。

易混點：正規化 (normalization) ≠ 正則化 (regularization)。 前者調數值尺度（本節）；後者防過擬合（L1/L2、dropout，主題 7）。中文很像，別搞混。

1.6 特徵值與特徵向量 — 變換下「不轉向」的軸

對方陣 $\mathbf{A}$，若存在非零向量 $\mathbf{v}$ 使 $\mathbf{A}\mathbf{v}=\lambda\mathbf{v}$，則 $\mathbf{v}$ 是特徵向量、純量 $\lambda$ 是特徵值。

名詞澄清。 這裡的「特徵」是 eigen（固有、特性），與 1.5 的「特徵 = feature（輸入變數）」只是中文撞名、毫無關係。尤其注意：本節的特徵向量 (eigenvector) ≠ 資料的 feature vector（1.1 的 $\mathbf{x}$）。判斷靠上下文 — 講矩陣／變換／PCA／伸縮就是 eigen，講資料／輸入／欄位就是 feature。

幾何意義。 矩陣＝對空間做線性變換（旋轉 / 拉伸 / 剪切）；大多數向量被 $\mathbf{A}$ 一乘會轉向，而特徵向量是少數「方向不變、只被拉長或縮短」的特殊軸，倍率就是 $\lambda$（$\lambda \gt 1$ 拉長、$0 \lt \lambda \lt 1$ 縮短、$\lambda \lt 0$ 反向、$\lambda=1$ 不變）。把單位圓上所有向量用 $\mathbf{A}$ 變換會變成橢圓 — 橢圓的長短軸方向＝特徵向量、半軸長＝特徵值。

與 AI 的關聯（重點）。 這不是純數學；ML 常透過以下方式用到它：

用途特徵值 / 特徵向量的角色
PCA 主成分分析 ★對資料共變異數矩陣做特徵分解：特徵向量＝變異最大的「主方向」、特徵值＝該方向的變異量 → 降維、抽特徵、去冗餘、視覺化
資料形狀描述資料雲（橢圓）的軸向與胖瘦
譜分群 / 圖 (GNN)圖 Laplacian 的特徵向量 → 譜分群、圖神經網路理論

最常見的入口是 PCA：網路遙測常有數十個彼此相關的特徵，PCA 用特徵分解找出「真正有變化的少數主方向」，把維度降下來、去掉冗餘 — 這就是特徵值 / 向量在 ML 的日常用途，也和主題 8 Embedding 的「用少數維度表徵資料」一脈相承。

先搞懂「共變異數矩陣 (covariance matrix)」是什麼。 它由兩個更基本的量組成（符號：$x_i,y_i$＝第 $i$ 筆資料在兩個特徵（feature）上的值，非 eigenvalue；$\bar x,\bar y$＝兩特徵的平均，$\bar x=\frac1n\sum_i x_i$；$n$＝資料點數／樣本數）：

  • 變異數 $\text{Var}(x)=\frac{1}{n-1}\sum_i(x_i-\bar x)^2$：單一特徵「散多開」。
  • 共變異數 $\text{Cov}(x,y)=\frac{1}{n-1}\sum_i(x_i-\bar x)(y_i-\bar y)$：兩特徵「是否一起動」。關鍵看乘積的正負 — 某點的 $x,y$ 同在各自平均的同一側（都偏大或都偏小）→ 正；不同側 → 負。平均後：Cov $\gt$ 0 正相關（一起漲）、Cov $\lt$ 0 負相關（此消彼長）、$\approx$ 0 無關。

把它們排成表就是共變異數矩陣（$d$ 個特徵 → $d\times d$）：對角線 = 各特徵的變異數、非對角線 = 兩兩共變異數；因 $\text{Cov}(x,y)=\text{Cov}(y,x)$ 而對稱：

$$\mathbf{C}=\begin{bmatrix}\text{Var}(x) & \text{Cov}(x,y)\\ \text{Cov}(x,y) & \text{Var}(y)\end{bmatrix}$$

（除以 $n-1$ 是樣本變異數的標準做法 — 貝索校正，因平均 $\bar x$ 由同批資料估出、會讓離差平方和略偏小；若把資料當成整個母體則除以 $n$。）它把資料雲的形狀與傾斜編碼成一個矩陣（對角線 = 各方向散多開、非對角線 = 往哪個斜方向相關）；對它做特徵分解就得到雲的主軸 — 這就是下面的 PCA。

動手做一次 PCA（數字例子）。 用 5 個 2D 點示範「共變異數矩陣做特徵分解」的完整流程。資料：$(5,4),(4,5),(3,3),(2,1),(1,2)$。

  1. 中心化：平均 $=(3,3)$，減掉後得 $(2,1),(1,2),(0,0),(-1,-2),(-2,-1)$。
  2. 共變異數矩陣：分別算 $\text{Var}(x)$、$\text{Var}(y)$、$\text{Cov}(x,y)$ 再填進矩陣（用中心化後的 $x,y$，除以 $n-1=4$）：$\text{Var}(x)=\frac{2^2+1^2+0+(-1)^2+(-2)^2}{4}=\frac{10}{4}=2.5$；$\text{Var}(y)=\frac{1^2+2^2+0+(-2)^2+(-1)^2}{4}=2.5$；$\text{Cov}(x,y)=\frac{(2)(1)+(1)(2)+0+(-1)(-2)+(-2)(-1)}{4}=\frac{8}{4}=2$（← 這裡才同時用到 $x$ 與 $y$）： $$\mathbf{C}=\begin{bmatrix}\text{Var}(x)&\text{Cov}(x,y)\\ \text{Cov}(x,y)&\text{Var}(y)\end{bmatrix}=\begin{bmatrix}2.5 & 2\\ 2 & 2.5\end{bmatrix}$$ （緊湊寫法：$\mathbf{C}=\frac{1}{n-1}\sum_i(\mathbf{x}_i-\bar{\mathbf{x}})(\mathbf{x}_i-\bar{\mathbf{x}})^\top$，其中 $\mathbf{x}_i=(x_i,y_i)^\top$ 是整筆資料的向量、兩邊是同一個向量；外積時乘法會把 $x$ 分量配 $y$ 分量，自動在非對角線生出 $x{\cdot}y$ 共變異數項 — 所以「兩邊都是 $\mathbf{x}$」是對的，$y$ 藏在向量裡。）
  3. 特徵分解：解特徵方程得 $\lambda_1=4.5,\ \lambda_2=0.5$，對應特徵向量 $\mathbf{v}_1=\frac{1}{\sqrt2}(1,1)$、$\mathbf{v}_2=\frac{1}{\sqrt2}(1,-1)$（計算細部見下）。
  4. 解讀：$\mathbf{v}_1=(1,1)$ 是變化最大的主軸（變異 $\lambda_1=4.5$），佔總變異 $\frac{4.5}{4.5+0.5}=90\%$；$\mathbf{v}_2$ 垂直於它、變異僅 0.5。
  5. 投影降維：把中心化點投影到 $\mathbf{v}_1$（做內積，1.1）得 1D 座標 $\approx 2.12,\ 2.12,\ 0,\ -2.12,\ -2.12$ — 原本每點 2 個數字，現在 1 個數字就抓住 90% 的變化。

第 3 步（特徵分解）怎麼算？ 拆成三小步：

(a) 特徵值 — 解特徵方程。 由定義 $\mathbf{C}\mathbf{v}=\lambda\mathbf{v}$ 移項成 $(\mathbf{C}-\lambda\mathbf{I})\mathbf{v}=\mathbf{0}$；要有非零解 $\mathbf{v}$，就必須讓該矩陣「壓扁空間」（否則只有零解），即 $\det(\mathbf{C}-\lambda\mathbf{I})=0$（$\det$＝行列式，$=0$ 代表矩陣壓扁空間、不可逆；定義見 1.3）：

$$\det\begin{bmatrix}2.5-\lambda & 2\\ 2 & 2.5-\lambda\end{bmatrix}=(2.5-\lambda)^2-2^2=0 \;\Rightarrow\; 2.5-\lambda=\pm2 \;\Rightarrow\; \lambda_1=4.5,\ \lambda_2=0.5$$

(b) 特徵向量 — 每個 $\lambda$ 代回 $(\mathbf{C}-\lambda\mathbf{I})\mathbf{v}=\mathbf{0}$ 解。

  • $\lambda_1=4.5$：$\mathbf{C}-4.5\mathbf{I}=\begin{bmatrix}-2&2\\2&-2\end{bmatrix}$，方程 $-2v_1+2v_2=0\Rightarrow v_1=v_2$ → 取 $(1,1)$，正規化成 $\frac1{\sqrt2}(1,1)$。
  • $\lambda_2=0.5$：$\mathbf{C}-0.5\mathbf{I}=\begin{bmatrix}2&2\\2&2\end{bmatrix}$，方程 $2v_1+2v_2=0\Rightarrow v_1=-v_2$ → 取 $(1,-1)$，正規化成 $\frac1{\sqrt2}(1,-1)$。

(c) 驗證 $\mathbf{C}\mathbf{v}=\lambda\mathbf{v}$：

$$\mathbf{C}\begin{bmatrix}1\\1\end{bmatrix}=\begin{bmatrix}2.5+2\\2+2.5\end{bmatrix}=\begin{bmatrix}4.5\\4.5\end{bmatrix}=4.5\begin{bmatrix}1\\1\end{bmatrix},\qquad \mathbf{C}\begin{bmatrix}1\\-1\end{bmatrix}=\begin{bmatrix}0.5\\-0.5\end{bmatrix}=0.5\begin{bmatrix}1\\-1\end{bmatrix}$$

$\mathbf{C}$ 乘上去沒轉向、只縮放，倍率就是特徵值 — 完全符合 1.6「特徵向量＝不轉向的軸」。（$\lambda$ 從大排到小：$\mathbf{v}_1$ 是變化最大的第一主成分。）

第 5 步（投影降維）：2 個數字 → 1 個數字。 一個 2D 點要用兩個座標定位；投影到主軸 $\mathbf{v}_1$ 後，只需「沿 $\mathbf{v}_1$ 的位置」一個數（用內積算，1.1）：

原本（2 個數）投影到 $\mathbf{v}_1$（1 個數）
$(2,1)$$2.12$
$(1,2)$$2.12$
$(0,0)$$0$
$(-1,-2)$$-2.12$
$(-2,-1)$$-2.12$

例：$(2,1)\cdot\frac1{\sqrt2}(1,1)=\frac{3}{\sqrt2}\approx2.12$。注意 $(2,1)$ 與 $(1,2)$ 都變 2.12 — 它們沿 $\mathbf{v}_1$ 落在同位置、只差在被丟掉的 $\mathbf{v}_2$ 方向；分不出這兩點，就是那損失的 10%。若這 10% 不重要，用 1 個數取代 2 個數就很划算（真實場景：50 維壓到 5 維幾乎不失真）。

因為 $\mathbf{C}$ 對稱，特徵向量必互相垂直 → 主軸是一組正交新座標系（呼應 1.3.1 換座標系）。幾何上 $\mathbf{C}$ 把資料雲編碼成一個沿 $(1,1)$ 拉長的橢圓：長軸方向 = $\mathbf{v}_1$、半軸長² = 特徵值。

工程意涵。 「模型推論」在數學上 ≈ 一串矩陣乘法。模型大小（參數量）≈ 所有 $\mathbf{W}$ 的元素數；推論算力 ≈ 矩陣乘法次數 (FLOPs)。這直接對應「模型越大越吃 VRAM 與算力」。

2. 機率與統計 — 分布、條件機率、最大似然

判別式模型輸出「機率」、生成式模型學「分布」、訓練的損失又來自「最大似然」——所以這章是 AI 的統計地基。以下每個概念都給「定義 → 直覺/物理意義 → 在 AI 的用途」。

2.1 機率、隨機變數與分布

機率 (probability) — 介於 0～1 的數，衡量某事「有多可能」：0 不可能、1 必然、所有結果機率總和 = 1。直覺：長期做很多次時該事件出現的比例（公平硬幣正面 0.5）。用途：分類器輸出「這封信是垃圾郵件的機率 = 0.92」就是它。

隨機變數 (random variable) $X$ — 一個「值還不確定、依機率取值」的量：離散（可數，如骰子點數、類別）或連續（任意實數，如身高、流量 bytes）。

符號提醒。 大寫 $X$ = 隨機變數本身；小寫 $x$ = 它可能取到的某個具體值。$X=x$ 是一個事件（「$X$ 取到值 $x$」），所以 $P(X=x)$ 讀成「$X$ 等於 $x$ 的機率」——整個 "$X=x$" 是括號內描述的事件（不是 $P(X)=x$）。例：公平骰子 $P(X=3)=\frac16$。

機率分布 (distribution) — 「機率怎麼分配到 $X$ 各個值」的概念。要把它具體寫下來，依變數型態用不同的描述函數：

  • 離散 → PMF（機率質量函數） $P(X=x)$：給每個值一個機率，且所有值機率加總 = 1：$\sum_x P(X=x)=1$（像長條圖）。
  • 連續 → PDF（機率密度函數） $p(x)$：連續值有無限多、單點機率為 0，改看「密度」；某區間的機率 = 曲線下面積 $\int_a^b p(x)\,dx$，且全部面積 = 1：$\int_{-\infty}^{\infty} p(x)\,dx=1$（像一條曲線）。
  • 通用 → CDF（累積分布函數） $F(x)=P(X\le x)$：「小於等於某值的機率」，離散、連續都適用。

直覺：PMF、PDF、CDF 都是描述同一個分布的不同表達；分布就是「機率怎麼分配到各個值」的地圖，越高處值越常出現。下圖是離散分布用 PMF 描述的例子（公平骰子）：

期望值 $\mathbb{E}[X]$ — 分布的「中心／平均」：

$$\mathbb{E}[X]=\sum_x x\,P(x)\ (\text{離散})\qquad \mathbb{E}[X]=\int x\,p(x)\,dx\ (\text{連續})$$

物理意義：分布的重心（把機率當質量，期望＝平衡點），也是「長期平均值」。用途：平均損失、平均獎勵 (RL)、平均表現都是期望。

例（等機率就是算術平均）：公平骰子 $\mathbb{E}[X]=\frac{1+2+3+4+5+6}{6}=3.5$——注意 3.5 不是骰子會出現的值；期望是「長期平均／重心」，不必是某個可能結果。

期望的線性性（超好用）：$\mathbb{E}[X+Y]=\mathbb{E}[X]+\mathbb{E}[Y]$（不論 X、Y 是否獨立）。配合指示變數 $\mathbb{1}_A$（事件 A 發生記 1、否則 0，$\mathbb{E}[\mathbb{1}_A]=P(A)$），就能把「數某種東西的期望個數」拆成一堆機率相加——這是組合數學/機率計算的核心技巧。在 AI：整批資料的平均損失就是逐筆損失期望的線性相加，所以能用小批次 (mini-batch) 估計整體損失。

變異數 $\mathrm{Var}(X)=\mathbb{E}[(X-\mathbb{E}[X])^2]$、標準差 $\sigma=\sqrt{\mathrm{Var}(X)}$ — 「散開程度」： 物理意義是資料離中心多遠（平方後平均）；$\sigma$ 小＝擠在平均附近、大＝散得開（與 1.6 的變異數同一件事）。用途：衡量不確定性、資料尺度（正規化 1.5）、風險。下圖以最常見的常態（高斯）分布示意：

2.2 條件機率、獨立與貝氏定理

條件機率 $P(A\mid B)$ — 「已知 B 發生，A 的機率」：

$$P(A\mid B)=\frac{P(A\cap B)}{P(B)}$$

直覺：多了一個已知條件，把考慮範圍縮小到「B 發生的世界」裡再看 A 的比例。例：「已知信裡有『中獎』兩字，它是垃圾郵件的機率」遠高於「隨便一封信是垃圾郵件」——條件改變了機率。用途：分類器算的就是 $P(\text{類別}\mid\text{特徵})$。

獨立 (independence)。 若 $P(A\mid B)=P(A)$（知不知道 B 都不影響 A），則 A、B 獨立、$P(A\cap B)=P(A)P(B)$。用途：樸素貝氏 (Naive Bayes) 假設特徵彼此獨立來簡化計算。

貝氏定理 (Bayes) — 用證據更新信念：

$$P(y\mid x)=\frac{P(x\mid y)\,P(y)}{P(x)}$$

直覺：$P(y)$ 是「看資料前的先驗信念」，看到證據 $x$ 後更新成「後驗」$P(y\mid x)$。例：某病盛行率低（先驗低），檢測陽性後患病機率被往上修，但仍受先驗牽制——這就是罕見病偽陽性要小心的原因。

  • 判別式模型直接學 $P(y\mid x)$ — 分類器就在估這個。
  • 生成式模型學 $P(x)$、$P(x\mid y)$（資料如何產生）— 所以能「生成」，也可用貝氏反推分類。

這正是主題 0 判別 vs 生成分野的數學根。

2.3 似然與最大似然估計 (MLE)

先分清「機率」與「似然」（同一式子、兩種看法）：

  • 機率：固定參數，問「資料出現的可能性」。
  • 似然 (likelihood)：固定觀測到的資料，問「哪組參數最能解釋這些資料」。把 $P(\text{資料}\mid\theta)$ 看成 $\theta$ 的函數，就叫似然。

最大似然 (MLE) — 「訓練」在統計上多半就是：找一組 $\theta$ 讓實際觀測到的資料出現機率最大（＝最能解釋資料的參數）。

$$\theta^\star=\arg\max_\theta \prod_{i=1}^{N} P(y_i\mid x_i;\theta)$$

直覺：調整模型旋鈕，直到「模型覺得真實資料很正常、很可能出現」。連乘不好算 → 取對數變連加（log 單調、不改最大點）、加負號變最小化：

$$\theta^\star=\arg\min_\theta\; -\sum_{i=1}^{N}\log P(y_i\mid x_i;\theta)$$

這個 $-\sum\log P$ 就是負對數似然，也就是分類最常用的交叉熵損失（主題 5）。損失函數多半是負對數似然的化身，不是憑空設計。

工程意涵。 分類器輸出的是機率/信心分數而非硬標籤；資安可用它調「靈敏度 vs 誤報」門檻（主題 19 ROC）。懂 MLE 就懂為什麼交叉熵是預設損失。

3. 微積分與梯度 — 導數、偏導、鏈鎖律

訓練 = 最小化損失 $L(\theta)$。怎麼往更小走？靠梯度。

3.1 導數、偏導與梯度

導數 $\dfrac{dL}{d\theta}$ — 變化率（切線斜率）。 定義：$\theta$ 動一點點時 $L$ 跟著變多少的比率（$\lim_{\Delta\theta\to0}\frac{\Delta L}{\Delta\theta}$，即切線斜率）。物理意義：斜率為正 → $L$ 上升、為負 → 下降、為 0 → 平地（駐點：谷底/山頂/鞍點）。用途：它告訴我們「往哪個方向調、調多少」能降低損失。

偏導 $\dfrac{\partial L}{\partial\theta_j}$ — 多變數時「只動一個旋鈕」的變化率（固定其他參數，看這一個對 $L$ 的單獨影響）。

梯度 $\nabla_\theta L$ — 把所有偏導疊成一個向量：

$$\nabla_\theta L=\left(\frac{\partial L}{\partial\theta_1},\frac{\partial L}{\partial\theta_2},\dots\right)^\top$$

幾何意義：梯度指向「$L$ 上升最快」的方向、長度＝那個方向的陡度；要下降就往反方向走 — 這是梯度下降（主題 6）的全部直覺：

$$\theta\leftarrow\theta-\eta\,\nabla_\theta L$$

把損失想成一個「碗／山谷」，站在坡上某點：梯度是最陡上坡、$-\nabla L$ 是最陡下坡；每步走 $\eta$（學習率）大小，逐步滾向谷底。$\eta$ 太大會衝過頭、太小會很慢。

3.2 鏈鎖律 — 反向傳播的引擎

鏈鎖律 (chain rule) — 複合函數的「影響力連乘」。 神經網路是層層複合 $L=f_k(f_{k-1}(\cdots f_1(\mathbf{x})))$：$w$ 先影響 $z$、$z$ 影響 $a$、$a$ 影響 $L$。直覺：$w$ 對 $L$ 的總影響率＝沿路每一段影響率相乘（像齒輪傳動、一環扣一環）：

$$\frac{dL}{dw}=\frac{dL}{da}\cdot\frac{da}{dz}\cdot\frac{dz}{dw}$$

例如 $L=(y-\hat y)^2$、$\hat y=\sigma(z)$、$z=wx+b$，則

$$\frac{\partial L}{\partial w}=\underbrace{2(\hat y-y)}_{\partial L/\partial\hat y}\cdot\underbrace{\sigma'(z)}_{\partial\hat y/\partial z}\cdot\underbrace{x}_{\partial z/\partial w}$$

反向傳播就是把鏈鎖律由輸出往輸入逐層套用（主題 10 完整推導）。局部導數在前向時記下、反向時相乘，讓百萬參數的梯度一次算出。

工程意涵。 「訓練吃算力遠大於推論」的數學原因：推論只做前向（一遍矩陣乘法）；訓練要前向 + 反向（再算梯度）+ 更新，且重複很多輪。對應「邊緣做推論、再訓練放中央」。

4. 線性回歸與損失函數

最小的「會學習」模型，把前三章串起來。

4.1 模型與損失

線性回歸假設 $\hat y=\mathbf{w}^\top\mathbf{x}+b$。用均方誤差 (MSE) 當損失：

$$L(\mathbf{w},b)=\frac{1}{N}\sum_{i=1}^{N}\big(\hat y_i-y_i\big)^2=\frac{1}{N}\sum_{i=1}^{N}\big(\mathbf{w}^\top\mathbf{x}_i+b-y_i\big)^2$$

MSE 其實是「誤差服從高斯分布」下的最大似然（主題 2）— 再次印證損失來自機率假設。

幾何意義。 在散點圖上找一條直線，讓每個點到線的「垂直距離」平方和最小。垂直距離就是殘差 $\hat y_i - y_i$；平方是為了懲罰大誤差、且對應高斯假設。

分類 vs 回歸（一張圖看懂）。 同一堆資料點：分類是拿超平面把點雲「切開」（判在哪一側/哪一區）；回歸是拿一條線「穿過」點雲（讓點盡量貼近、預測數值）。兩者都建立在 $\mathbf{w}^\top\mathbf{x}+b$ 上，差別只在輸出怎麼用 — 分類看落在哪一側（argmax），回歸直接讀數值。

4.2 梯度與更新

$$\nabla_{\mathbf{w}}L=\frac{2}{N}\sum_{i=1}^{N}\big(\hat y_i-y_i\big)\,\mathbf{x}_i$$

再用梯度下降 $\mathbf{w}\leftarrow\mathbf{w}-\eta\nabla_{\mathbf{w}}L$。「預測誤差 $(\hat y-y)$ 乘輸入 $\mathbf{x}$」的形式會在神經網路反覆出現 — backprop 最外層永遠是誤差項。

工程意涵。 網路 AI 的「頻寬/健康預測」本質就是回歸；線性回歸是理解一切的起點。MSE 適合連續值，不適合分類（下節換損失）。

5. 邏輯回歸、Sigmoid、Softmax 與交叉熵

分類模型的數學核心 — 幾乎所有分類器（含神經網路輸出層）都用這套。

5.1 二元分類：Sigmoid + 二元交叉熵

要輸出機率 (0~1)，把線性輸出 $z=\mathbf{w}^\top\mathbf{x}+b$ 過 sigmoid：

$$\sigma(z)=\frac{1}{1+e^{-z}}\in(0,1)$$

幾何意義。 Sigmoid 是一條 S 形曲線，把整條實數線「壓」進 (0,1)：$z=0$ 時輸出 0.5（決策臨界），$z$ 很大趨近 1、很小趨近 0。剛好可當「屬於正類的機率」。

$\hat y=\sigma(z)$ 解讀為 $P(y=1\mid x)$。損失用二元交叉熵 (BCE)（即負對數似然）：

$$L=-\frac{1}{N}\sum_{i=1}^{N}\big[y_i\log\hat y_i+(1-y_i)\log(1-\hat y_i)\big]$$

漂亮處：$\sigma'(z)=\sigma(z)(1-\sigma(z))$，且 BCE 對 $z$ 的梯度化簡成 $\frac{\partial L}{\partial z}=\hat y-y$ — 又是「預測減真值」，乾淨、數值穩定。

5.2 多元分類：Softmax + 交叉熵

$C$ 類時，輸出層給 $C$ 個分數 (logits) $\mathbf{z}=(z_1,\dots,z_C)$，用 softmax 轉成機率分布：

$$\text{softmax}(\mathbf{z})_k=\frac{e^{z_k}}{\sum_{j=1}^{C} e^{z_j}}$$

各項 0~1 且總和為 1。損失用類別交叉熵（$y$ 為 one-hot、真類為 $t$）：

$$L=-\sum_{k=1}^{C} y_k\log\hat y_k=-\log\hat y_t$$

即「真類機率的負對數」— 越確信正確類、損失越小。梯度同樣化簡為 $\frac{\partial L}{\partial z_k}=\hat y_k-y_k$。

幾何意義（多類怎麼「切」）。 二元分類是「一個超平面切成兩邊」；多類不是連續二分，而是 $C$ 個分數各自比高低、取最大（argmax）：

  • 每個 logit $z_k=\mathbf{w}_k\cdot\mathbf{x}+b_k$ 是 $\mathbf{x}$ 對「第 $k$ 類樣板 $\mathbf{w}_k$」的對齊程度（接回 1.1）；分類＝挑最對齊的類：$\hat y=\arg\max_k z_k$。
  • 幾何上這把空間切成 $C$ 個區域：每點歸給分數最高的類；相鄰兩類的邊界是「分數相等 $z_i=z_j$」的超平面。線性 softmax 時區域為凸多邊形、邊界為直線段。
  • 前面幾層若已用 $\sigma$ 摺疊過空間（1.3.1），這些邊界在原始空間就能任意彎曲，切出任何形狀、任意多類。

5.3 數值穩定：softmax 減 max

$e^{z_k}$ 在 $z$ 大時溢位。實作先減最大值（結果不變）：

$$\text{softmax}(\mathbf{z})_k=\frac{e^{z_k-\max_j z_j}}{\sum_j e^{z_j-\max_j z_j}}$$

工程意涵。 Softmax 是所有分類器與 LLM 輸出層的共同終點 — 網路流量分類輸出「各類別機率」、LLM 輸出「下一個 token 的機率分布」用的都是它。懂 softmax + 交叉熵，等於掌握判別式分類與生成式取樣（主題 17）的共同基座。

7. 正則化 (Regularization) — 對抗過擬合

主題 6（梯度下降／優化器）待補；本節先建立正則化要對付的問題 — 過擬合。

7.1 過擬合 vs 欠擬合

過擬合 (overfitting)：模型把訓練資料「背過頭」，連資料裡的雜訊、巧合都學進去 → 訓練集分數超高、但遇到沒看過的新資料就差。它學到的是「這批資料的特例」，不是「通則」。就像學生死背考古題，換個題目就不會。

欠擬合 (underfitting)：反過來，模型太簡單，連訓練資料的趨勢都抓不到 → 訓練、測試都差。

模型複雜度幾何訓練測試
欠擬合太簡單直線硬套彎曲資料、抓不到趨勢差差
剛好適中平滑曲線貼合趨勢、容許每點有點誤差好好
過擬合太複雜扭曲曲線硬穿每個點、連雜訊都學極好差

怎麼判斷：訓練誤差低、但驗證／測試誤差高，兩者落差 (gap) 大 = 過擬合的信號。原因通常是模型太複雜（參數太多）、資料太少、或訓練太久。

怎麼防（＝正則化的任務）：讓模型「別把訊號記太死」——

  • L1 / L2 正則化：懲罰權重大小 $\lVert\mathbf{w}\rVert$（用 1.4 的範數球限制權重），逼模型用更小／更少的權重。
  • 其他：更多資料、early stopping（早點停）、dropout、簡化模型、交叉驗證。

工程意涵。 Demo 時「訓練準確率 99%」可能只是背了那批樣本，換真實流量就失效——這就是過擬合。驗收邊緣模型要用按時間切、含真實比例惡意樣本的測試集，看它對「沒看過的資料」還準不準。

7.2 要訓練幾輪 (epoch)？看驗證損失

一個 epoch = 把整份訓練資料完整跑一遍（前向→損失→反向→更新）。一遍學不會——梯度下降每次只走一小步，要重複多輪損失才逐步降下來、模型收斂。但不是越多輪越好：訓練損失隨 epoch 一路下降；驗證損失（沒看過的資料）先降後升——觸底那一輪就是最佳輪數，之後再練就開始過擬合。

怎麼決定 — Early stopping（早停）：一邊訓練一邊監看驗證損失，連續 N 輪（patience）沒進步就停、並保留「驗證損失最低」的存檔。它自動決定輪數，本身也是一種正則化。太少 epoch 欠擬合、太多 epoch 過擬合。

12. CNN 卷積

影像若用「全連接」硬做：一張 224×224×3 的圖攤平就是 15 萬維，第一層每個神經元就要 15 萬個權重——參數爆炸，還把「相鄰像素才相關」「同物件換位置仍是同物件」這些影像特性全丟了。卷積 (convolution) 就是為影像量身打造的解法。

12.1 卷積在做什麼（定義）

一個卷積核 (filter / kernel) 是一個小方陣（如 3×3）的權重。它滑過整張影像，在每個位置把 filter 蓋在對齊的小塊上，做逐元素相乘再加總（＝一次內積，§1.1）得到一個數；所有位置的數排起來就是一張特徵圖 (feature map)：

$$\text{特徵圖}(i,j)=\sum_{u,v}\text{filter}(u,v)\cdot\text{影像}(i{+}u,\,j{+}v)$$

注意：一步卷積 = 內積（一個數），不是矩陣×矩陣（後者結果是矩陣）。

12.2 幾何意義：filter 是「樣板」、特徵圖是「對齊強度圖」

接回 §1.1（內積＝對齊程度）與 §1.2（神經元＝特徵偵測器）：

  • filter 就是一個「局部樣板」（小塊空間裡的一個方向向量）。
  • 每個位置的卷積＝量「這一小塊有多像這個樣板」（對齊程度 / 投影）——越像、數值越大（激活越強），不像則接近 0 或負。
  • 所以整張特徵圖 = 「這個樣板在影像各處出現的強度分佈圖」（像一張 heat map）。

例：一個「垂直邊樣板」filter 掃過影像後，特徵圖會在有垂直邊的地方變亮、平坦處接近 0——等於幫你標出「圖裡哪裡有垂直邊」。這就是卷積的幾何本質：一個到處掃描的特徵偵測器。

淺層 filter 學到邊緣 / 角 / 紋理等低階樣板，多層堆疊後組合成部件、物件等高階樣板——就是「在越來越抽象的樣板空間裡量對齊」（呼應 §1.3.1 逐層換座標）。

12.3 三個關鍵性質（為什麼卷積適合影像）

  • 局部連接（感受野）：每個輸出只看一小塊，符合「相鄰像素才相關」。
  • 參數共享：同一個 filter 掃全圖——不管物件在哪都用同一組權重 → 參數量大減（3×3 filter 只有 9 個權重，卻掃遍全圖）。
  • 平移不變：物件移到別處，同一 filter 仍抓得到 → 認得出「換位置的同一物件」。
  • 多通道：輸入 RGB 有 3 通道、filter 也有對應深度；用多個 filter 就得多張特徵圖（每張抓一種樣式）。

12.4 stride、padding 與輸出尺寸

  • stride（步幅）：filter 每次滑幾格（越大、輸出越小）。
  • padding（補邊）：在影像邊緣補 0，用來保住尺寸或照顧邊界像素。
  • 輸出邊長 $=\dfrac{W-F+2P}{S}+1$（$W$ 輸入邊長、$F$ filter 邊長、$P$ padding、$S$ stride）。

12.5 整張卷積 = 一次矩陣乘法（im2col）

雖然一步是內積，整張卷積可重排成一次大矩陣乘法：把每個小塊攤平成一列、堆成大矩陣 $\mathbf{X}_{\text{patches}}$，filter 攤平成 $\mathbf{W}$，一次 $\mathbf{X}_{\text{patches}}\mathbf{W}^\top$（GEMM）就算出整張特徵圖。這叫 im2col，也是 GPU/NPU 加速卷積的方法（§20）——所以「卷積本質是矩陣乘法」指的是這個整體，不是那兩個小方陣直接相乘。

工程意涵。 卷積不只用在影像：1D 卷積掃封包大小/方向序列，可抓「加密流量的局部指紋」（網路 AI 常用、輕量、適合邊緣 NPU）。多數邊緣 NPU 本來就為 CNN 卷積最佳化，所以 1D/2D-CNN 在邊緣跑很划算。

20.1 GPU/NPU 為什麼能加速 AI 計算

核心一句話：神經網路的前向與反向（微積分）幾乎全部化約成矩陣乘法 (matmul) + 逐元素運算，這兩種運算「大量、獨立、可同時做」——GPU/NPU 就是為此而生。

為什麼矩陣運算天生可平行。 矩陣乘法 $\mathbf{C}=\mathbf{A}\mathbf{B}$，每個輸出 $C_{ij}=$（A 第 $i$ 列）$\cdot$（B 第 $j$ 欄）是一個獨立內積，彼此不相干 → 可同時算。CPU 少數強核心偏序列；GPU 有數千個小核心，一次做數千個「乘加 (MAC)」，把大量獨立內積塞滿。

GPU 的加速手段：

手段作用
數千個 ALU同時做數千個乘加
Tensor Cores專用硬體，一個時脈算完一小塊矩陣乘
高頻寬記憶體 HBMTB/s 級頻寬餵大量矩陣資料
分塊 (tiling) + 片上 SRAM切小塊放快取重複用，少去慢速主記憶體
低精度 FP16/BF16/INT8每時脈算更多、頻寬省 → 吞吐倍增

NPU/TPU 更專精、更省電：

  • 脈動陣列 (systolic array)：一整片 MAC 單元排成網格，資料像水流過、每個單元邊流邊乘加 → 一次吞一大塊矩陣乘，能效比 GPU 更高。
  • 幾乎整顆晶片都是乘加器、多以 INT8/INT4 定點運算為主 → 低功耗高吞吐，正適合邊緣設備（代價是通用性低）。

反向傳播的「微積分」也是矩陣乘法。 以一層 $\mathbf{z}=\mathbf{W}\mathbf{x}$ 為例（鏈鎖律，3.2）：

$$\text{前向：}\ \mathbf{z}=\mathbf{W}\mathbf{x}\qquad \text{反向：}\ \frac{\partial L}{\partial \mathbf{x}}=\mathbf{W}^\top\frac{\partial L}{\partial \mathbf{z}},\quad \frac{\partial L}{\partial \mathbf{W}}=\frac{\partial L}{\partial \mathbf{z}}\,\mathbf{x}^\top$$

反向兩式又都是 matmul；激活函數的導數是逐元素運算。所以「訓練 = 前向 matmul + 反向 matmul + 逐元素」，全是 GPU/NPU 的強項；自動微分 (autodiff) 就是把微分自動拆成這串可平行運算。

工程意涵。 ① 訓練算力 ≫ 推論：推論只做前向一遍，訓練要前向＋反向（再一整套 matmul）＋重複多輪 → 對應「邊緣做推論、再訓練放中央」。② 參數量 → matmul 次數 (FLOPs) → 算力與記憶體需求（幾 B 模型為何吃資源）。③ 量化 (INT8/INT4) 讓 NPU 吞吐↑、記憶體↓ → 邊緣落地的關鍵。

附錄 A — 餘弦定律 (Law of Cosines)

內積的代數定義與幾何定義為何相等 (主題 1.1)？關鍵是餘弦定律。本附錄先給餘弦定律的敘述 (A.1) 與自足推導 (A.2，不使用內積)，再用它證明內積兩種定義等價 (A.3)。

A.1 敘述

三角形三邊長 $a, b, c$，設 $C$ 為邊 $c$ 所對的夾角，則

$$c^2 = a^2 + b^2 - 2ab\cos C$$

  • 是畢氏定理的推廣：當 $C=90^\circ$，$\cos C=0$ → $c^2=a^2+b^2$（畢氏定理）。餘弦定律多了修正項 $-2ab\cos C$。
  • $C \lt 90^\circ$（銳角）→ $\cos C \gt 0$ → $c^2 \lt a^2+b^2$；$C \gt 90^\circ$（鈍角）→ $\cos C \lt 0$ → $c^2 \gt a^2+b^2$。夾角越大，對邊越長。

A.2 推導（座標法，最乾淨）

把夾角 $C$ 放在原點、邊 $b$ 沿 x 軸：$C=(0,0)$、$A=(b,0)$、$B=(a\cos C,\ a\sin C)$。

邊 $c=\lvert AB\rvert$，用距離公式：

$$c^2 = (b - a\cos C)^2 + (0 - a\sin C)^2$$

展開：

$$c^2 = b^2 - 2ab\cos C + a^2\cos^2 C + a^2\sin^2 C$$

因 $\cos^2 C + \sin^2 C = 1$（畢氏恆等式），$a^2\cos^2 C + a^2\sin^2 C = a^2$：

$$\boxed{\;c^2 = a^2 + b^2 - 2ab\cos C\;}\qquad\blacksquare$$

此推導只用了距離公式 + 畢氏恆等式，完全獨立於內積，所以下面 A.3「用餘弦定律證內積」不構成循環論證。（等價作法：從 $B$ 對邊 $b$ 作垂線、用畢氏定理分段相加，結果相同。）

A.3 用餘弦定律證明內積的兩種定義等價

內積有兩個定義 — 代數的「分量乘積之和 $\sum_i a_ib_i$」與幾何的「$\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi$」。用 A.2 的餘弦定律可證它們相等。取由 $\mathbf{a}$、$\mathbf{b}$ 與其差 $\mathbf{a}-\mathbf{b}$ 圍成的三角形（夾角 $\phi$）：

(1) 代數展開（純用分量、畢氏）：

$$\lVert\mathbf{a}-\mathbf{b}\rVert^2=\sum_i (a_i-b_i)^2=\sum_i a_i^2-2\sum_i a_ib_i+\sum_i b_i^2=\lVert\mathbf{a}\rVert^2+\lVert\mathbf{b}\rVert^2-2\sum_i a_ib_i$$

(2) 餘弦定律（A.2，三邊 $\lVert\mathbf{a}\rVert$、$\lVert\mathbf{b}\rVert$、$\lVert\mathbf{a}-\mathbf{b}\rVert$，夾角 $\phi$）：

$$\lVert\mathbf{a}-\mathbf{b}\rVert^2=\lVert\mathbf{a}\rVert^2+\lVert\mathbf{b}\rVert^2-2\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi$$

兩式左邊同為 $\lVert\mathbf{a}-\mathbf{b}\rVert^2$，右邊相等；消去 $\lVert\mathbf{a}\rVert^2+\lVert\mathbf{b}\rVert^2$ 得：

$$\boxed{\;\sum_i a_ib_i=\lVert\mathbf{a}\rVert\lVert\mathbf{b}\rVert\cos\phi\;}$$

即「分量乘積之和」恰等於「兩向量長度 × 夾角餘弦」 — 代數與幾何兩種內積定義其實是同一件事，這也解釋了為什麼「分量硬乘相加」會等於「量測方向對齊程度」。
